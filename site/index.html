Live security · agentic systems

Find a vulnerability, exploit it, then watch it get closed.

A self-contained security lab that spins up a per-visitor, network-isolated environment on demand — a deliberately vulnerable web target beside an attacker shell, usable right in your browser and auto-destroyed after 30 minutes. The entire stack is provisioned as code, so the public attack surface stays deliberately small even though the lab itself runs intentionally vulnerable software. A docked AI coach teaches each challenge end to end — from the exploit to the fix.

What happens when you start

01

Isolated per session

Each visitor gets a dedicated Docker network with no internet egress. Sessions can't reach each other or phone home.

02

Target + attacker shell

A real vulnerable app (OWASP Juice Shop) in an iframe, beside a live shell box you work from — wired together on the private network.

03

Verifiable objective

Choose a challenge and solve it. A server-side check reads the target's own state to confirm the exploit landed — no self-reporting.

04

Ephemeral by design

Containers are resource-capped, capability-dropped, and reaped on a 30-minute timer. Nothing carries over between sessions.

05

Coached by an AI guide

A docked chat, scoped to your challenge and real progress, teaches both the exploit and the fix — server-side, and declining anything aimed at real systems.

06

Exploit, then remediate

On the SQL-injection challenge you break in, then the lab applies the fix to the running target and replays the exploit to prove the hole is closed.

How it's built

The public surface is deliberately tiny. Caddy terminates TLS and serves the site, reverse-proxying the API, the target iframe, and the shell WebSocket to a Node orchestrator that listens on localhost only. The orchestrator drives the Docker API to create, proxy, and reap per-session containers on internal no-egress networks. Everything — network, instance, DNS, web server, container images, and the orchestrator — is defined in Terraform and reproduced from a single apply, with remote state in S3 and separate dev and prod environments.

Where it's going

The exploit-to-remediation loop is live; next is hardening it for the long run: production monitoring (availability, certificate validity, anomaly and exposure checks, per-IP rate limiting), an automated test suite and CI that gate every change, and challenge authoring that scales — security shown end to end, and kept continuously verified.
